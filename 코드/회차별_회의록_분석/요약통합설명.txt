기본적으로 수집된 회의록 데이터의 모든 회차 폴더를 순회하여 데이터를 읽어옵니다.

folders = [f for f in os.listdir(data_dir) if f.startswith("제") and f.endswith("회")]
folders.sort(key=lambda x: int(x.replace("제", "").replace("회", "")))

위 코드는 data_dir 내부의 제353회, 제354회와 같은 회차 폴더를 자동으로 탐색한 뒤, 회차 번호 기준으로 정렬하여 순차적으로 처리합니다.

각 회차에서 추출된 데이터는 작업 목적에 맞게 작성된 프롬프트와 함께 OpenAI API를 사용하여 요약 또는 정리 작업을 수행합니다.

세 개의 코드 파일은 전체적인 처리 구조는 동일하며, 회의록 데이터를 순회하면서 OpenAI API를 이용해 분석 결과를 생성하는 방식으로 구성되어 있습니다.

코드 간 차이는 **사용되는 프롬프트와 GPT에 전달되는 텍스트 단위(회차 단위 또는 발언자 단위)**에 있으며, 이를 통해 다음과 같은 서로 다른 결과를 생성합니다.

회차별 회의 내용 요약

요구자료 목록 추출

발언자별 발언 요약

처리가 완료된 결과는 아래와 같이 CSV 파일로 저장됩니다.

output_path = os.path.join(DATA_DIR, "요구자료_목록_379_414회_통합.csv")
result_df.to_csv(output_path, index=False, encoding="utf-8-sig")
주의사항

기존 작성자의 작업 방식으로 보았을 때 20대·21대·22대 국회를 각각 필요한 범위의 데이터만 다운로드한 뒤 코드를 실행하고, 생성된 CSV 파일의 이름을 수동으로 변경하여 관리한 것으로 보입니다.

따라서 모든 회차 데이터를 한 번에 수집한 뒤 실행하더라도 코드는 정상적으로 동작할 가능성이 있습니다. 다만 다음과 같은 사항에 주의가 필요합니다.

실행 중 OpenAI API 토큰 부족 등으로 작업이 중단될 가능성이 있습니다.

현재 코드에는 중간 결과를 저장하거나 이어서 실행하는 백업 로직이 포함되어 있지 않습니다.

따라서 다음과 같은 방법 중 하나를 권장합니다.

회차 범위를 나누어 여러 번에 나누어 실행

충분한 API 토큰을 확보한 상태에서 실행

또는 중간 결과를 저장하는 백업 코드를 추가하여 실행
